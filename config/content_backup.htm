進入 cad2019g0 倉儲中的 downloads 目錄, 將 scrum-1 帳號下的 cad2019 納為 downloads/scrum1 子模組

cd downloads

利用 git submodule add https://github.com/scrum-1/cad2019.git  scrum1

 然後利用 git submodule update --init --recursive 將所有子模組更為最新版本

假如只是要 git clone 且連同所有原設定版次的子模組, 且至多同時擷取 8 個子模組內容, 使用

git clone --recurse-submodules -j8 https://github.com/mdecourse/cp2019g0.git

將 https://github.com/scrum-1/cad2019 設為位於 downloads 目錄下的子模組, 且名稱訂為 scrum1

之後各組員可以在各自的 cp2019 倉儲中建立相關輔助設計程式或零件, 然後直接在 cad2019g0 的環境下引用.

為何各組員的 cad2019 作為子模組, 需要放在分組網站倉儲 downloads 目錄下?

因為 https://github.com/mdecourse/cmsimde/blob/master/flaskapp.py 中的 372 行所宣告的 downloads 函式, 會將其下的動態網站資料路徑送回瀏覽器, 而且轉為靜態網頁時, 882 行的 get_page2 函式會將 downloads 下資料連結的絕對目錄轉為相對目錄.

近端編輯:

cad0.bat

@echo off
y:
cd tmp\cad2019g0\cmsimde
start /WAIT python wsgi.py | sleep 2 | start chrome https://localhost:9443
cd \
cls
